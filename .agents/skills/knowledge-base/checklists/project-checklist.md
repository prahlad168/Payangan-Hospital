# Project Development Checklist

Use this checklist for every task or project completion.

## Pre-Development

- [ ] Understand requirements clearly
- [ ] Identify dependencies
- [ ] Set up development environment
- [ ] Create branch for development
- [ ] Review existing similar projects
- [ ] Check knowledge base for existing SOPs

## During Development

- [ ] Follow coding standards
- [ ] Document as you go
- [ ] Test locally before commit
- [ ] Keep commits atomic and descriptive
- [ ] Update progress tracker
- [ ] Take screenshots for documentation

## Pre-Deployment

- [ ] Review all changes
- [ ] Test in staging (if available)
- [ ] Verify all links work
- [ ] Check mobile responsiveness
- [ ] Verify image optimization
- [ ] Run /play command for QA

### Quality Assurance (/play)

- [ ] Agent 1: Link Checker passed
- [ ] Agent 2: QA Checker passed
- [ ] Agent 3: Content Validator passed
- [ ] Agent 4: SEO Optimizer passed
- [ ] Agent 5: Performance passed
- [ ] Agent 6: Accessibility passed
- [ ] Agent 7: UI/UX Enhancer passed
- [ ] Agent 8: Image Optimizer passed
- [ ] Agent 9: Code Quality passed
- [ ] Agent 10: Mobile Responsive passed
- [ ] Agent 11: Security Checker passed
- [ ] Agent 12: Content Freshness passed
- [ ] Agent 13: Deployment Ready passed

## Deployment

- [ ] Push to GitHub
- [ ] Verify webhook triggers
- [ ] Confirm hosting updated
- [ ] Test live site
- [ ] Verify all functionality

## Post-Deployment

- [ ] Document lessons learned
- [ ] Update SOP if needed
- [ ] Update knowledge base
- [ ] Share with relevant teams
- [ ] Monitor for errors
- [ ] Gather feedback

## Knowledge Extraction

After completion, create:
- [ ] Lessons Learned document
- [ ] Updated SOP (if process improved)
- [ ] New templates or prompts (if applicable)
- [ ] Best Practice document (if applicable)

For quick daily tasks:

- [ ] Understand task
- [ ] Execute task
- [ ] Test result
- [ ] Document outcome
- [ ] Update if needed
